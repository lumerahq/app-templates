Payroll Journal Entry

Upload payroll reports, AI generates debit/credit journal lines, review for balance and post.

What This App Does

Every pay cycle, payroll providers generate reports — PDFs or CSVs — that need to be translated into journal entries for your general ledger. This app automates that translation.

Upload a payroll report and AI reads it, figures out the correct debit/credit lines with account codes, departments, and memos, and presents them for review. The user checks that debits equal credits, makes any corrections, and posts the entry.

  • Handles PDFs and CSVs — The AI automation works with both formats
  • Balance validation — The detail page highlights when total debits ≠ total credits
  • GL account management — Maintain a chart of payroll GL accounts in settings
  • Dashboard — View payroll runs by status with totals and recent activity

Payroll Run Lifecycle

draft → processing → review → posted or rejected
  1. Upload — User uploads a payroll report (PDF or CSV). A payroll_runs record is created with the pay period dates, pay date, and attached document. Status: draft.
  2. Auto-trigger — The trigger_extract hook fires on after_create, sets status to processing, and queues the extract_payroll automation.
  3. AI extraction — The automation downloads the document, reads it with AI, and generates journal entry lines. Each line gets an account code, account name, department, debit amount, credit amount, and memo. The automation creates journal_entries records and computes totals. Status moves to review.
  4. Review — The detail page shows the original document alongside the journal entry table. Totals are displayed at the bottom. If debits ≠ credits, the imbalance is highlighted.
  5. Post or reject — User clicks Post to finalize the entries, or Reject if corrections are needed.

Balance validation: The detail page compares total_debits vs total_credits. If they don't match, the imbalance is highlighted in red. The user should correct journal lines before posting.

Automations & Hooks

extract_payroll automation

The core AI automation. Takes a run_id, downloads the attached report, reads it with llm.extract_text(), and generates journal entry lines. It handles both PDF and CSV formats — the AI determines the format and parses accordingly. After creating all journal_entries records, it computes the debit/credit totals and updates the payroll run.

External ID: payroll-journal-entry:extract_payroll

trigger_extract hook

Fires on payroll_runs after_create. Sets status to processing and queues the extract_payroll automation. This makes the AI extraction fully automatic on upload.

Key Design Decisions

  • Journal entries as separate records: Each debit/credit line is its own record in journal_entries, linked to the payroll run via a relation field. This makes balance calculations and per-department breakdowns straightforward.
  • Both PDF and CSV support: The AI automation handles both formats. PDFs are read with vision, CSVs are parsed as text. The same automation handles both.
  • Totals stored on the run: total_debits and total_credits are computed by the automation and stored on the payroll run for quick dashboard display without re-summing.
  • No auto-post: The app always stops at review status. A human must explicitly post, ensuring the journal entry is verified before it's finalized.
