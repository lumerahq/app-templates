What This App Does

This is a minimal starting point for building a Lumera app. It comes with:

  • A dashboard with placeholder stat cards and a recent activity section
  • An example collection (example_items) to show how data is structured
  • A settings page as a placeholder for configuration
  • A seed script that populates demo data
  • An AI agent (assistant) ready to be customised

The idea is to replace example_items with your own collections, build real pages, and add automations and hooks as needed.

How a Lumera App Fits Together

Every Lumera app follows the same pattern: a React frontend runs inside an iframe in the Lumera platform, talking to the Lumera API for data and automation. All backend resources are defined as code in the platform/ directory and deployed with the CLI.

Key concept: The platform/ directory is the source of truth. Edit collection schemas, automations, hooks, and agents there, then deploy with lumera apply.

Building on This Template

The typical pattern for adding a new feature:

  1. Define the collection — Add a JSON schema in platform/collections/
  2. Build the UI — Add route files in src/routes/, query data with pbSql or pbList
  3. Add automation (if needed) — Create a Python script in platform/automations/ for AI processing or background work
  4. Wire up hooks (if needed) — Add JS hooks in platform/hooks/ to trigger automations on create/update events
  5. Seed data — Update scripts/seed-demo.py with realistic test records
  6. Deploy — Run lumera plan then lumera apply
Tip: Read the skill files in .claude/skills/ for detailed API docs on collections, automations, hooks, and agents.
