Fixed Asset Tracker

Track assets, run monthly depreciation, view schedules, and handle disposals with gain/loss.

What This App Does

This app manages the full lifecycle of fixed assets — from acquisition to disposal. It maintains an asset register, calculates monthly depreciation (straight-line or declining balance), tracks accumulated depreciation and net book value, and records disposals with gain/loss calculations.

  • Asset register — Track equipment, vehicles, software, furniture, buildings, and leasehold improvements with cost basis, useful life, and salvage value
  • Depreciation automation — Run depreciation for a single asset or all active assets for a given period. Supports straight-line and declining-balance methods
  • Depreciation schedule — Each asset's detail page shows a full schedule of past depreciation entries
  • Disposal tracking — Record disposal date and proceeds, automatically compute gain/loss vs. net book value
  • Dashboard — Total net book value, asset counts by status, accumulated depreciation, and category breakdown
  • GL accounts — Maintain asset, contra-asset, and expense accounts in settings

Asset Lifecycle

Assets start as active and can end up in one of two final states:

  • active → fully_depreciated — Automatic. When the depreciation automation reduces the net book value to the salvage value, it sets the status automatically. No user action needed.
  • active → disposed — Manual. User clicks "Dispose", enters the disposal date and proceeds, and the system records the gain/loss.

Depreciation Flow

This is the core monthly operation. The user picks a period (YYYY-MM) and either runs depreciation for a single asset or triggers a batch run for all active assets.

  1. User clicks "Run Depreciation" from the asset detail page or the batch depreciation page
  2. The calculate_depreciation automation runs for the given asset and period
  3. It computes the monthly depreciation amount based on the asset's method (straight-line: (cost − salvage) / useful_life_months, declining balance: NBV × 2 / useful_life_months)
  4. A depreciation_entries record is created with the amount, new accumulated total, and resulting NBV
  5. The asset's accumulated_depreciation is updated
  6. If NBV has reached the salvage value, status is automatically set to fully_depreciated

Disposal Flow

When an asset is retired or sold, the user records the disposal from the asset detail page:

  1. User clicks "Dispose" and enters the disposal date and proceeds amount
  2. The app computes gain/loss: proceeds − net_book_value
  3. The asset record is updated with disposal date, proceeds, gain/loss, and status is set to disposed

Automations

calculate_depreciation automation

The sole automation in this app. Takes two inputs:

  • asset_id (string) — which asset to depreciate
  • period (string, YYYY-MM) — which month to calculate for

It fetches the asset, computes the monthly amount based on the depreciation method, creates a depreciation_entries record, updates the asset's accumulated_depreciation, and checks whether NBV has reached salvage value (if so, auto-sets fully_depreciated).

External ID: fixed-asset-tracker:calculate_depreciation

Idempotency: The automation should check if an entry already exists for this asset + period before creating a duplicate. The seed script may have already created entries for demo data.

Key Design Decisions

  • No hooks in this template: Unlike invoice-processing and payroll, there's no automatic trigger on asset creation. Depreciation is always user-initiated (monthly batch or per-asset), which is the standard accounting workflow.
  • Two depreciation methods: Straight-line ((cost − salvage) / months) and declining balance (NBV × 2 / months). The method is per-asset, set at creation time.
  • Disposal is a simple update: No separate automation — the frontend computes gain/loss client-side and updates the asset record directly. This keeps it simple since disposal is a one-time event per asset.
  • Period as text, not date: The period field on depreciation entries is stored as YYYY-MM text, not a date. This makes it easy to query and prevents confusion about which day of the month.
  • Depreciation entries have their own status: Entries start as pending and can be marked posted. This allows a review step before finalizing depreciation in the ledger.
